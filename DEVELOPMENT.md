# Development Guide

## Настроенные инструменты

### ESLint

- Конфигурация находится в `eslint.config.mjs`
- Проверяет TypeScript и JavaScript код
- Автоматически исправляет некоторые проблемы

### Prettier

- Конфигурация находится в `.prettierrc`
- Автоматически форматирует код при сохранении
- Игнорирует файлы согласно `.prettierignore`

## Доступные команды

### Линтинг

```bash
# Проверить код
npm run lint

# Автоматически исправить проблемы
npm run lint:fix
```

### Форматирование

```bash
# Отформатировать код
npm run format

# Проверить форматирование
npm run format:check
```

### Сборка и тестирование

```bash
# Компиляция
npm run compile

# Тесты
npm run test

# Полная проверка перед тестами
npm run pretest
```

## Настройки VS Code

Проект включает настройки VS Code для автоматического:

- Форматирования при сохранении (Prettier)
- Исправления проблем ESLint при сохранении
- Валидации TypeScript и JavaScript

### Рекомендуемые расширения

- Prettier - Code formatter
- ESLint
- TypeScript Importer

## Конфигурация

### Prettier

- Полужирные кавычки
- Точка с запятой в конце
- Ширина строки: 100 символов
- Отступ: 2 пробела

### ESLint

- Строгие правила для TypeScript
- Проверка качества кода
- Автоматическое исправление доступных проблем

## Игнорируемые файлы

Следующие файлы и папки исключены из проверки и форматирования:

- `node_modules/`
- `dist/` и `out/`
- Конфигурационные файлы
- Изображения и бинарные файлы
